# Contributing

Thanks for your interest in contributing to `pi-web-tools`!

This repo follows the [agents.md convention](https://agents.md/) and keeps its
contributor guide in [`AGENTS.md`](./AGENTS.md). The PR, branch, and commit
conventions apply equally to human contributors.

## Where to look

- **PR / branch / commit conventions:** [`AGENTS.md`](./AGENTS.md)
- **Build / test / format commands:** [README → Development](./README.md#development)
- **Security issues:** [`SECURITY.md`](./SECURITY.md)

## One rule worth restating

**Every PR must reference a pre-existing GitHub issue, and one PR delivers one issue.**
Open the issue first, link it from the PR. See [`AGENTS.md`](./AGENTS.md) for the full
policy (including how to granulate an issue that's too big for a single PR).
